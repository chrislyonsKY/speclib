Methodology

Spectral processing methods used in the speclib viewer and Python package.

1. Data Sources and Ingestion

speclib federates reflectance spectra from multiple public spectral libraries through dedicated ingestion adapters. Each adapter preserves original metadata and provenance information without modification.

1.1 USGS Spectral Library Version 7 (splib07a)

The primary source for mineral, vegetation, soil, artificial material, organic compound, coating, and liquid spectra. Data is ingested from the ASCII distribution of splib07a, which provides reflectance measurements from multiple spectrometers:

InstrumentRangeChannelsType
ASD FieldSpec FR/HR/NG0.35 – 2.5 μm2,151VNIR-SWIR
Beckman 52700.2 – 3.0 μm480UV-SWIR
Nicolet FTIR (NIC4)1.12 – 216 μm4,595SWIR-TIR
AVIRIS 19960.37 – 2.5 μm224Imaging spectrometer

Reflectance values of -1.23e+34 in the USGS data are treated as no-data and excluded from all calculations. Wavelength arrays are instrument-specific and paired to each spectrum based on the instrument code in the filename.

Kokaly, R.F., Clark, R.N., Swayze, G.A., et al., 2017, USGS Spectral Library Version 7: U.S. Geological Survey Data Series 1035, 61 p., https://doi.org/10.3133/ds1035.

1.2 Kentucky Field Data (Lyons, Gyawali, Cristan, Acharya, Gebremedhin, & Andries, 2024)

Field-collected leaf reflectance spectra targeting invasive species detection in Kentucky recreation areas. Measurements were collected using a CID CI-710 leaf spectrometer with an Ocean Optics USB4000 sensor (345 – 1,035 nm, 3,648 channels).

The collection includes Lonicera maackii (Amur honeysuckle) leaf reflectance across three canopy strata (upper, mid, lower) from multiple individual plants at Raven Run and Shelby Trails sites in central Kentucky.

Lyons, W. C., Gyawali, B. R., Cristan, R., Acharya, S., Gebremedhin, M., & Andries, K. (2024). Evaluating spectral properties of invasive plant species in Kentucky recreation areas. International Journal of Remote Sensing, 45(18), 6470–6494. https://doi.org/10.1080/01431161.2024.2391095.

1.3 Naming Convention

All spectra follow a USGS-inspired naming convention:

Material_Name SampleCode.MeasurementSuffix [Descriptor]

USGS examples: Actinolite HS116.3B, Quartz GDS31 0-74um fr

Kentucky field data: L.maackii KY01.U1 Leaf where KY01 is the individual plant, U1 is upper canopy replicate 1, and Leaf is the tissue type.

2. Spectral Units and Conventions

  • Wavelength: micrometers (μm), always sorted ascending
  • Reflectance: unitless ratio, 0.0 – 1.0 (not percent)
  • Spatial reference (KY data): EPSG:3089 (Kentucky Single Zone, NAD83)
  • No-data: stored as null in JSON, NaN in arrays; excluded from all computations
Reflectance values are stored as-measured. No interpolation, smoothing, or resampling is applied during ingestion. All spectral transforms in the viewer are computed client-side from original data.

3. Continuum Removal

Continuum removal normalizes a reflectance spectrum by dividing it by its spectral continuum (the convex hull upper envelope). This isolates absorption features and enables comparison of absorption depth independent of overall albedo.

3.1 Algorithm

  1. Compute the upper convex hull of the spectrum in wavelength–reflectance space using Andrew's monotone chain algorithm.
  2. Interpolate the convex hull vertices linearly to produce a continuous continuum line at every wavelength.
  3. Divide the original reflectance by the continuum at each wavelength.
RCR(λ) = R(λ) / C(λ)

Where RCR is continuum-removed reflectance, R is original reflectance, and C is the interpolated convex hull continuum.

Values where C(λ) < 0.001 are set to 1.0 to avoid division by near-zero.

Continuum-removed reflectance ranges from 0.0 to 1.0, where 1.0 indicates the spectrum touches its continuum (no absorption) and values < 1.0 indicate absorption features. This is the standard approach used in USGS spectroscopy analysis.

Clark, R.N. and Roush, T.L., 1984, Reflectance spectroscopy: Quantitative analysis techniques for remote sensing applications: Journal of Geophysical Research, v. 89, p. 6329–6340.

4. Spectral Derivatives

Spectral derivatives enhance subtle features and inflection points that are difficult to identify in raw reflectance data. The first derivative is particularly useful for identifying the red edge position in vegetation spectra (~0.68–0.75 μm).

4.1 First Derivative

Computed using central finite differences:

dR/dλi = (Ri+1 − Ri−1) / (λi+1 − λi−1)

4.2 Second Derivative

Computed by applying the central difference operator to the first derivative:

d²R/dλ²i = (dR/dλi+1 − dR/dλi−1) / (λi+1 − λi−1)

No smoothing or filtering is applied prior to differentiation. For noisy spectra, consider applying a Savitzky-Golay filter in the Python package before derivative analysis.

Tsai, F. and Philpot, W., 1998, Derivative analysis of hyperspectral data: Remote Sensing of Environment, v. 66, p. 41–51.

5. Band Indices

The viewer computes standard spectral indices by interpolating reflectance at the required band center wavelengths. Indices are computed from the continuous library spectrum, not from convolved sensor bands.

IndexFormulaBands (μm)Application
NDVI (NIR − Red) / (NIR + Red) NIR: 0.865, Red: 0.655 Vegetation vigor
NDWI (Green − NIR) / (Green + NIR) Green: 0.56, NIR: 0.865 Water content
NDSI (Green − SWIR1) / (Green + SWIR1) Green: 0.56, SWIR1: 1.61 Snow/ice
EVI 2.5 × (NIR − Red) / (NIR + 6×Red − 7.5×Blue + 1) NIR: 0.865, Red: 0.655, Blue: 0.48 Enhanced vegetation

Reflectance at each band center is obtained by linear interpolation between the two nearest measured wavelengths. If a required wavelength falls outside the spectrum's range, the index returns N/A.

NDSI will return N/A for spectra that do not extend to the SWIR1 region (1.61 μm), such as the Kentucky field data measured with the CID CI-710 (345 – 1,035 nm).

6. Spectral Convolution to Sensor Bands

The convolution preview shows what a satellite sensor would measure by convolving the library spectrum with each sensor band's spectral response function (SRF).

6.1 Method

Each sensor band is modeled as a Gaussian spectral response function centered at the band's nominal wavelength with FWHM equal to the band's spectral width:

SRF(λ) = exp(−0.5 × ((λ − λcenter) / σ)²)
where σ = FWHM / 2.355

The convolved reflectance for each band is then:

Rband = ∑ SRF(λi) × R(λi) / ∑ SRF(λi)

6.2 Supported Sensors

SensorBands ShownRange
Landsat 8/9 OLICoastal, Blue, Green, Red, NIR, SWIR1, SWIR20.43 – 2.29 μm
Sentinel-2 MSIB1–B8A, B11, B120.44 – 2.19 μm

Convolved values appear as discrete marker points on the chart when the Convolve toggle is enabled alongside a sensor band overlay.

Markham, B.L. and Barker, J.L., 1985, Spectral characterization of the Landsat Thematic Mapper sensors, International Journal of Remote Sensing, v. 6, p. 697–716.

7. Group Statistics

When multiple spectra are loaded, the Stats button computes a mean spectrum and ±1 standard deviation envelope, displayed as a shaded band behind the individual traces.

7.1 Method

  1. Determine the common wavelength range shared by all loaded spectra.
  2. Resample all spectra to the first trace's wavelength grid within the common range using linear interpolation.
  3. At each wavelength, compute the arithmetic mean and population standard deviation across all traces.
μ(λ) = (1/n) × ∑ Ri(λ)
σ(λ) = √((1/n) × ∑ (Ri(λ) − μ(λ))²)

The envelope shows μ ± 1σ, which contains approximately 68% of observations for normally distributed data. This is useful for characterizing intra-species spectral variability.

8. Data Quality and Provenance

8.1 Quality Tiers

TierDescriptionCriteria

CURATEDExpert-reviewed field measurementsPeer-reviewed publications with documented collection protocols
COMMUNITYCommunity contributionsSubmitted via contribution workflow, basic quality checks passed

8.2 Provenance Tracking

Every spectrum in speclib retains:

  • Source library and original record identifier
  • Instrument used for measurement
  • Measurement type (absolute reflectance, relative reflectance, etc.)
  • License and citation information
  • Ingestion timestamp and adapter version

Metadata is never modified during ingestion. Missing fields are stored as null, never fabricated.
